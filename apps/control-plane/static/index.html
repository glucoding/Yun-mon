Release Notes

发布说明

  • 保存后会统一生成 `.env`、Prometheus、Alertmanager、Loki、Promtail 与 Grafana 控制台入口配置。
  • Prometheus 会热重载；文件型配置变更可通过“重启监测运行时”让现有容器重新加载。
  • 当 `stack-agent` 可用时，控制台会走宿主机侧 `docker compose up -d --build`，把端口、环境变量和镜像重建类变更一并闭环。
  • 当 `stack-agent` 不可用时，会自动回退到容器级 Docker API 重启；默认不会重启当前 `control-plane`。

定义监测平台的项目标识、部署环境和集群名称，用于统一标签与环境区分。

Platform Configuration

外部端口

管理监测组件对宿主机暴露的访问端口，决定控制台、Grafana 和各服务的入口地址。

Platform Configuration

Grafana

负责展示仪表盘、日志检索和告警看板，是用户查看监测结果的主要界面。

Platform Configuration

Prometheus

负责采集指标、执行规则计算并向告警链路输出监测事件，是指标监测的核心引擎。

Platform Configuration

Alertmanager

负责对 Prometheus 告警进行分组、抑制和路由，是告警治理与通知编排中心。

Platform Configuration

Loki

负责集中存储和查询日志数据，与 Grafana 联动实现日志检索与故障排查。

Platform Configuration

Promtail

负责从应用日志文件中采集日志并推送到 Loki，是日志接入链路的采集代理。

Platform Configuration

Docker Stats Exporter

补充采集 Docker 容器的 CPU、内存、网络和磁盘指标，弥补 cAdvisor 标签不足场景。

Platform Configuration

cAdvisor

采集主机与容器运行时的基础资源指标，是容器资源监测的底层数据来源之一。

Platform Configuration

Host Stack Agent

运行在宿主机侧，负责执行真正的 docker compose build 与重建，让控制台具备完整发布闭环。
